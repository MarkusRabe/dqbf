What this is

Provers and tooling for Dependency Quantified Boolean Formulas built around the fork resolution proof system. A DQBF asserts the existence of Boolean functions with restricted argument lists; the goal is to extract those functions (Skolem functions, emitted as AIGER circuits), not just decide SAT/UNSAT.

Architecture:

core/          shared Formula IR, DQDIMACS, AIGER, proof-trace format
provers/       one dir per prover; forkres/ = Python ref + rust/
tools/eqfob/   EQFOB language: BV + ∃-quantified functions → DQBF
tools/verify/  independent checkers (trusted base; imports only core/)
benchmarks/    holdout/ (eval-only) | train/ (scalable, generated)
               runner/ (parallel harness, compare)

Tools

CLIdoes
forkresDQDIMACS → SAT/UNSAT/UNKNOWN; --cert .aag, --proof .frp
eqfob compileEQFOB source → DQDIMACS; -D NAME=INT overrides widths
dqbf-verify satDQDIMACS + .aag → DIMACS CNF (UNSAT⇒valid) + var-map JSON; --solve runs a SAT solver
dqbf-verify unsatDQDIMACS + .frp → VALID/INVALID (self-contained replay)
dqbf-bench runparallel run over a family; JSONL out + summary table
dqbf-bench comparebaseline.jsonl vs candidate.jsonl → Δsolved/regressions; exit 1 on reject

Example workflow

1. EQFOB source

-- ∃f,g : bv[A]→bv[B]. ∃z:bv[B]. ∀x:bv[A]. f(x) + z > g(x)
-- Satisfiable: pick g ≡ 0, z = 1, f arbitrary.
param A = 4
param B = 4

fun f : bv[A] -> bv[B]
fun g : bv[A] -> bv[B]

exists z : bv[B]
forall x : bv[A]

f(x) + z > g(x)
